# 高保真生成与拆件规则

## 目标

基于用户确认的 Zero 低保真稿生成高保真参考图，并完成视觉拆件和 Zero 可编辑重建。

## 前置条件

1. Zero 低保真稿已经由用户确认。
2. 已读取低保真稿的页面结构、槽位、文案和交互关系。
3. 需要先检查 `games-component-specs`，确认是否已有可复用标准组件。

## 生成规则

1. 先生成完整高保真视觉图，用于确认整体风格、构图、色彩和材质。
2. 高保真图只能作为参考层，不能作为最终唯一交付物。
3. 页面结构必须尽量保持与低保真稿一致。
4. 用户确认高保真方向后，再进入拆件与重建。

## 拆件规则

至少判断以下类型：

- 干净背景氛围图：移除主体、UI、文字和前景装饰后补全背景。
- 主 IP 或主要角色：单独生成或精细抠图。
- 核心玩法主体：保留原角度、材质、光线和安全边距。
- 前景装饰：需要调整位置、层级或动效时必须独立输出。
- 光效、粒子、纹理：优先输出透明 PNG。
- 文字、按钮、卡片：不得从整图抠取，必须在 Zero 中重建。

## 组件复用规则

1. 红包、京豆及其他已登记标准组件直接从 Zero 组件库复用。
2. 不从高保真 JPG 中抠取标准组件。
3. 不使用图像模型重新生成已有标准组件。
4. 仅当 PRD 明确要求全新非标准造型，且现有组件无法满足时，才作为新视觉素材处理。

## 建议素材命名

```text
01_background_clean.png
02_main_ip.png
03_core_game_object.png
04_reward_assets.png
05_foreground_decoration.png
06_light_effect.png
07_full_visual_reference.jpg
```

## 验收规则

1. 干净背景无 UI、文字、主体残影和明显补图断层。
2. 独立主体轮廓完整，无缺边、白边或背景污染。
3. 拆件组合后与完整高保真参考图构图基本一致。
4. 所有需要交互、换文案或换状态的元素均已在 Zero 中重建。
5. 无法可靠拆出的元素已明确标记处理方式。
