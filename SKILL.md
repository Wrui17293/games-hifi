# 高保真生成、拆件与重建规范

执行本规范时，先读取 `references/rules.md`，再进行高保真生成、视觉拆件和 Zero 可编辑重建。

基于用户已确认的 Zero 低保真稿生成高保真参考图，完成视觉拆件，并继续在 Zero 中进行组件化重建。

## 执行流程
1. 确认 Zero 低保真稿已由用户确认。
2. 读取低保真稿的页面结构、槽位、文案和交互关系。
3. 生成完整高保真视觉图，用于确认整体风格、构图、色彩和材质。
4. 建立视觉拆件清单，判断每个元素采用单独生成、抠图、背景补全或 Zero 重建。
5. 按拆件清单生成或提取视觉素材。
6. 将完整高保真图放入 Zero，作为锁定参考图层。
7. 导入拆件素材，并在 Zero 中重建文字、按钮、卡片和可点击入口。
8. 使用组件或可复用 Frame 整理重复元素。
9. 最终返回 Zero 高保真可编辑链接、视觉拆件素材和组件拆解清单。

## 视觉拆件清单

生成整张高保真图后，至少判断以下内容：

| 类型 | 推荐处理方式 |
|---|---|
| 干净背景氛围图 | 移除主体、UI 和前景装饰后补全背景 |
| 主 IP／主要角色 | 参考整图单独生成或精细抠图 |
| 核心玩法主体 | 根据本次 PRD 的主题与玩法，单独生成或抠图，保留原角度、材质和光线 |
| 已有 Zero 标准组件 | 直接复用组件，不从高保真整图拆件 |
| 非标准奖励或主题素材 | 无可用组件时，才单独生成或抠图 |
| 前景装饰 | 单独生成或抠图 |
| 光效、粒子、纹理 | 单独生成透明素材，或使用适合后期处理的纯色背景 |
| 文字、按钮、卡片 | 不从整图抠取，必须在 Zero 中重建 |

建议文件命名：

```text
01_background_clean.png
02_main_ip.png
03_core_game_object.png
04_reward_assets.png
05_foreground_decoration.png
06_light_effect.png
07_full_visual_reference.jpg
```

## 拆件方法

### 1. 干净背景

- 以完整高保真图作为参考。
- 移除角色、玩法主体、文字、按钮、卡片和前景装饰。
- 根据周围画面补全被遮挡区域。
- 保持原图的构图、透视、光线方向、色彩和材质。
- 背景中不得残留文字、按钮轮廓或主体残影。

### 2. 主体物与 IP

- 需要后续移动、缩放、替换或制作动效的主要角色、主题物和核心玩法主体，应独立输出。
- 主体类型以本次 PRD 为准，不预设为宝箱、盲盒、转盘或其他固定主题。
- 优先参考整图单独生成，保持造型、角度、材质、颜色和光线一致。
- 能可靠抠图时可直接抠图；边缘污染、遮挡严重或缺失面积较大时应重新生成。
- 输出素材应保留完整轮廓、投影需求和足够安全边距。

### 3. 已有 Zero 标准组件

- 拆件前必须先读取 `games-component-specs`，检查是否已有可复用组件。
- 红包、京豆及其他已登记的标准组件直接从 Zero 组件库复用，不从高保真 JPG 中抠取，也不使用图像模型重新生成。
- 高保真参考图中的标准组件只用于确认位置、比例和视觉关系，不作为最终素材来源。
- 仅当 PRD 明确要求全新的非标准造型，且现有组件无法满足时，才作为新视觉素材处理，并标记为“待新增组件”。

### 4. 前景装饰与光效

- 彩带、云雾、植物、金币、粒子、光晕等应根据后续调整需求决定是否拆分。
- 需要改变位置、层级或动效的装饰必须独立输出。
- 与背景完全融合且无需移动的装饰可以保留在背景图中。
- 光效和粒子优先输出为透明 PNG；无法稳定生成透明素材时，使用便于后期处理的纯色背景。

### 5. UI 重建

以下内容不得依赖整张 JPG：

- 所有文字和数值。
- 主按钮、辅按钮和状态按钮。
- 卡片、标签、进度条和弹窗。
- 常驻功能信息入口。
- 资源位和 Feeds 信息结构。
- 所有需要点击、换文案、换状态或复用的元素。

这些内容必须根据高保真参考图在 Zero 中重建为可编辑图层或组件。

## 组件化判断规则

### 必须拆成可编辑组件的元素
- 所有文字
- 所有按钮
- 所有弹窗
- 所有卡片
- 所有可点击入口
- 所有奖励信息
- 所有商品/资源位
- 所有状态标签
- 所有需要后续复用或改文案的区域

### 可以保留为图片的元素
- 背景插画
- 复杂 3D 物体
- 光效、粒子、纹理
- 氛围装饰
- 不需要改文案、不需要复用的复杂视觉元素

### 必须独立拆件的图片元素
- 需要移动、缩放、替换或制作动效的 IP 和主体物
- 会遮挡其他组件、需要调整层级的前景装饰
- 无现成 Zero 组件且需要重复使用的非标准主题素材
- 需要单独控制透明度、混合方式或动画的光效
- 与点击区域或状态变化直接相关的视觉元素

以下元素不进入拆件：

- `games-component-specs` 中已有的 Zero 标准组件
- 可以直接在 Zero 中重建的文字、按钮、卡片和状态标签

## Zero 重建要求
- 高保真图只能作为参考层，不能作为最终唯一交付物。
- 文案必须重新创建为可编辑文本。
- 按钮、弹窗、卡片必须是独立图层或组件。
- 重复样式需要整理成组件变体，例如主按钮、次按钮、禁用按钮。
- 页面结构必须尽量保持与低保真稿一致。
- 如果某个视觉元素无法可靠拆解，需要标记为“保留图片资产”。

## 拆件验收

1. 干净背景中无 UI、文字、主体残影和明显补图断层。
2. 独立主体轮廓完整，无缺边、白边或背景污染。
3. 拆件组合后与完整高保真参考图的构图基本一致。
4. 所有需要交互、换文案或换状态的元素均已在 Zero 中重建。
5. 每个拆件命名清晰，并按背景、主体、装饰、光效和 UI 分类整理。
6. 无法可靠拆出的元素已明确标记处理方式，不伪装成可编辑组件。
